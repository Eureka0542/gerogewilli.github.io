第2章 Web概述

2.1 Java Web

使用java语言开发基于互联网的项目

2.2 软件框架

  1. C/S ：Client/Service 客户端/服务器端
    • 在用户本地有一个客户端程序，在远程有一个服务器端程序
    • 如：QQ、迅雷 等
    • 优点：用户体验好
    • 缺点：开发、安装、部署、维护 麻烦
  2. B/S： Browser/Service 浏览器/服务器端
    • 只需要通过浏览器访问不同的网址（URL），客户就可以访问不同的服务器端程序
    • 优点：开发、安装、部署、维护 简单
    • 缺点：如果应用过大，用户体验可能受影响；对硬件要求高

2.3 B/S架构的详解

  • 资源的分类
    • 静态资源、
      • 使用静态网页开发技术发布的资源
      • 特点：
        1. 所有用户访问得到的结果是一样的
        2. 如：文本、图片、音频、视频、HTML、CSS、JavaScript
        3. 如果用户请求的是静态资源，那么服务器会直接将静态资源发给浏览器。浏览器中内置的静态资源的解析引擎可以解析静态资源。
    • 动态资源
      • 使用动态网页开发技术发布的资源
      • 特点：
        1. 所有用户访问的到的结果可能不一样
        2. 如：jsp、servlet、php、sap
        3. 如果用户请求的是静态资源，那么服务器会执行动态资源，转换为静态资源，再发送给服务器

所以，在学习动态资源之前，先学习静态资源。

静态资源：

  1. HTML : 用于搭建基础网页，
  2. CSS：美化页面，布局页面
  3. JavaScript：控制页面的元素，让页面有一些动态的效果

        6.2 响应式布局

        • 同一套页面可兼容不同分辨率的设备。

        • 实现：依赖于栅格系统：将一行平均分成12个格子，可以指定元素占几个各自

        • 步骤：

          1. 定义容器。相当于之前的table
            1. container：固定宽度（两边有留白）
            2. container-fluid：每一种设备都是100%宽度
          2. 定义行。相当于之前的tr 样式：row
          3. 定义元素。指定该元素在不同的设备上，所占的格子数目。样式：col-设备代号-格子数目
            1. xs：超小屏幕 手机 （<768px) : col -xs-12
            2. sm: 小屏幕 平板 (≥769px)
            3. md：中等屏幕 桌面显示器（≥992px）
            4. lg：大屏幕 大桌面显示器 （≥1200px）

        • 注意事项：

          • 一行中如果格子数码超过12，超出部分自动换行。
          • 栅格类属性可以向上兼容。栅格类适用于屏幕宽度大于或等于分界点大小的设备。
          • 栅格类属性不能向下兼容。如果真实设备宽度小于设置栅格属性的设备代码的最小值，那么一个元素会占满一整行。

        第9章 HTTP

        9.1 概念: Hyper Text Transfer Protocol 超文本传输协议

        • 传输协议: 定义了客户端和服务器端交互信息时的数据格式
        • 特点:
          • 基于TCP/IP的高级协议
          • 默认端口号80
          • 基于请求/响应模型: 一次请求对应一次响应
          • 无状态: 每次请求之间相互独立,不能交互数据
        • 版本:
          • 1.0 : 每一次响应都会建立新的连接,响应完后断开连接
          • 1.1 : 复用连接。每一次响应后会等一会, 如果有新的请求，会复用上一次的连接，如果没有新的请求，则会断开连接。
